# Neovim 0.12.0

- Score: 277 | [HN](https://news.ycombinator.com/item?id=47565316) | Link: https://github.com/neovim/neovim/releases/tag/v0.12.0

## TL;DR

Neovim 0.12.0 is a new stable release with prebuilt binaries for all major platforms and a continued push toward “batteries-included” editing. The big direction change, highlighted by HN, is first‑party functionality like built-in LSP support and a new core plugin manager (`vim.pack`), which reduce reliance on sprawling starter packs. Discussion centers on how much should move from plugins into core, excitement about upcoming multiple cursors in 0.13, and what’s still missing before a 1.0 API-stable release.

---

## Comment pulse

- Fold more ecosystem ideas into core → fewer brittle plugins, easier “modern” setup; but core features must be long‑lived, stable APIs — counterpoint: plugins already work well.
- Multiple cursors are highly valued → in VS Code/Zed/Kakoune they replace many macro/find‑replace workflows and aid refactoring, despite Vim’s existing visual block and macro tools.
- New `vim.pack` manager → attractive as a standard built‑in layer; some still prefer higher‑level tools like lazy.nvim and note Vim already had basic packdir semantics.

## LLM perspective
