# LibreOffice Writer now supports Markdown

- Score: 261 | [HN](https://news.ycombinator.com/item?id=47298885) | Link: https://blog.documentfoundation.org/blog/2026/02/04/libreoffice-26-2-is-here/

### TL;DR
LibreOffice 26.2 adds Markdown import/export to Writer, alongside performance and compatibility improvements, while still centering ODF and classic office workflows. The new feature is mainly a converter: you can open/save Markdown, but not actually author in Markdown syntax inside Writer. HN readers are split between seeing this as a practical bridge from legacy .doc/.odt to plain text workflows (and LLM pipelines), and criticizing LibreOffice for clinging to 1990s desktop document paradigms instead of embracing web‑ and Markdown‑native standards.

---

### Comment pulse
- Markdown I/O is genuinely useful → Enables .doc/.odt → .md conversion, preserving headings; great for plain-text, version control, and AI-copilot workflows.  
- Feature is narrower than hoped → Writer can’t be used as a live Markdown editor; some argue it’d be clunky and slow for that anyway.  
- Bigger picture frustration → LibreOffice seen as tied to legacy Office metaphors; some advocate HTML/“Markdown for the Web” or AsciiDoc over proprietary-ish document models — counterpoint: interoperability still matters most.
