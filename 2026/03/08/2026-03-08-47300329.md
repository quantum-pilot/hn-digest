# Ask HN: Please restrict new accounts from posting

- Score: 398 | [HN](https://news.ycombinator.com/item?id=47300329) | Link: https://news.ycombinator.com/item?id=47300329

- TL;DR  
Hacker News is temporarily curbing Show HN posts from new accounts to fight spam, low-effort, and LLM-driven content. Many welcome more friction (proof-of-work, user muting, hiding) to restore signal, but warn that rigid account-age rules—like Reddit’s—mainly hurt sincere newcomers while bots adapt. A parallel debate targets LLM-written comments: some want bans for “obviously AI” text, others stress accessibility, editing help, and unreliable detection. Critics fear stricter controls will chill dissent and strengthen HN’s perceived hive-mind.

- Comment pulse
  - Limit new accounts, especially for Show HN → reduces bot/LLM noise and restores effort barrier—counterpoint: also blocks earnest first-time makers from sharing projects.
  - Reddit-style thresholds fail → spammers game karma/age checks while real users bounce; HN’s smaller scale and custom tooling might allow more nuanced, manual filters.
  - Crack down on obvious LLM comments → preserves human tone, avoids AI op-ed floods, but detection is unreliable and could punish assistive use and views.

- LLM perspective
  - View: Lightweight posting caps plus better per-user filters beat hard account-age rules or trying to detect AI text directly.
  - Impact: Newcomers still engage via replies; feed gets higher-signal posts; moderators focus on egregious abuse instead of gray-area LLM policing.
  - Watch next: Trials limiting Show HN to older accounts, basic quality metrics, and optional client features for muting submitters.
