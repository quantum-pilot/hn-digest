# A simple web we own

- Score: 164 | [HN](https://news.ycombinator.com/item?id=47124149) | Link: https://rsdoiel.github.io/blog/2026/02/21/a_simple_web_we_own.html

### TL;DR
- The author contends the web’s core tech is now cheap and simple enough that individuals and co‑ops can own their hardware, run tiny local “internets,” and publish via Markdown-driven static tools instead of relying on corporate platforms. This local-first, RSS-friendly model aims to escape surveillance capitalism and enshittified platforms while remaining compatible with the broader web. HN discussion stresses ISP and backbone control, discovery and attention bottlenecks, usability beyond Markdown, and the mixed role of corporations like GitHub.

### Comment pulse
- Infrastructure is the choke point → ISPs and hardware chains centralize power; without regulation, home servers have reach — counterpoint: mesh/community ISPs hint at workarounds.  
- Corporate hosting vs autonomy → some see using GitHub Pages as betraying the thesis; others note static sites remain portable, with corporations as interchangeable utilities.  
- Convenience and discovery dominate → hardware and CMS simplicity matter less than DNS, IPs, search rankings; RSS is remembered as lost, decentralized discovery layer.  

### LLM perspective
- View: Local-first publishing plus occasional cloud mirroring is pragmatic; ideological purity shouldn’t block people from using resilient infrastructure.  
- Impact: Easiest wins are tools bundling Markdown, RSS, search and hosting into one-click appliances for households, clubs and small organizations.  
- Watch next: Growth of community ISPs, mesh networks, AI-assisted editors, and standardized export formats that prevent lock-in across hosting providers.
