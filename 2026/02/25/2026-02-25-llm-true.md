# LLM=True

- Score: 202 | [HN](https://news.ycombinator.com/item?id=47149151) | Link: https://blog.codemine.be/posts/2026/20260222-be-quiet/

### TL;DR
The post argues that AI coding agents waste enormous context and tokens on noisy CLI output (build logs, colors, spinners, update notices), which degrades performance and costs money. Using a Turborepo/TypeScript example, the author shows how even “successful” builds flood Claude Code’s context, and how naive fixes like `| tail -N` cause repeated, slow reruns after failures. They propose a convention like `LLM=true` (analogous to `CI=true` or `NO_COLOR`) so tools can automatically emit shorter, cleaner, LLM-friendly logs—benefiting costs, quality, and energy use.
