# Grok Build is open source

- Score: 573 | [HN](https://news.ycombinator.com/item?id=48926590) | Link: https://github.com/xai-org/grok-build

- TL;DR
    - xAI has open‑sourced Grok Build, its Rust-based terminal coding‑agent harness and TUI, under Apache 2.0. The tool orchestrates LLMs to understand and edit codebases, run shell commands, search, and integrate with editors and CI via a headless mode and Agent Client Protocol. HN readers dig into unusual components like the Unicode Mermaid renderer and quickly produce privacy-focused and multi-provider forks. Discussion is split between enthusiasm for the smooth harness and skepticism after Grok’s code exfiltration and data‑retention controversies.

- Comment pulse
    - Impressive harness and model for some → users praise smooth Rust TUI and Mermaid renderer, Grok 4.5’s speed/quality, despite occasional instability and confusing keybindings.
    - Privacy/multi-provider forks emerge → Gork-Build and others strip telemetry, add provider choice, patches; seen as symbolic stance — counterpoint: critics expect these forks to wither.
    - Open-sourcing read as tactics, not altruism → commenters link it to Grok’s directory exfiltration scandal and Musk’s shaky trust “vibes,” undermining adoption.

- LLM perspective
    - View: This makes xAI’s agent harness a de facto reference for terminal-first coding agents and Agent Client Protocol implementations.
    - Impact: Open forks can pressure incumbents on privacy defaults, on-device options, and transparent data-retention controls, especially around whole-repo uploads.
    - Watch next: Watch for independent audits, multi-model plug-ins, and editor integrations; if they thrive, Grok Build’s scandal may fade behind its ecosystem.
