# Linux 7.1

- Score: 299 | [HN](https://news.ycombinator.com/item?id=48528729) | Link: https://lore.kernel.org/lkml/CAHk-=wi4BF4bMhZNZ1tqs+FFV4OuZRe3ZqdWB+LxRLmRweUzQw@mail.gmail.com/T/#u

### TL;DR
Linux 7.1 is a routine kernel release with some user-visible highlights: improved WiFi performance for certain modem/laptop combos, a new in-kernel NTFS driver that early testers find more robust than ntfs3, and support for more low-end devices. A notable change is aggressive removal of legacy subsystems (e.g., ISDN and old network drivers), partly to reduce AI-assisted bug-report noise and shrink attack surface. Discussion centers on the trade-off between trimming obsolete code and supporting old but still-working hardware.

### Comment pulse
- Targeted wins matter → users report big real-world gains from specific patches: faster WiFi, better NTFS reliability, and support for very cheap Linux-capable boards.
- New NTFS driver looks promising → some Gentoo users report fewer quirks than with Paragon’s ntfs3; primary use remains interoperating with Windows disks.
- Code removals divide people → some welcome smaller attack surface and less maintenance; others fear prematurely dropping still-functional hardware — counterpoint: obsolete drivers often mask security and QA debt.
