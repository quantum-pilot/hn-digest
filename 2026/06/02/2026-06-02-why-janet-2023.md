# Why Janet? (2023)

- Score: 422 | [HN](https://news.ycombinator.com/item?id=48367907) | Link: https://ianthehenry.com/posts/why-janet/

### TL;DR
Janet is a small, modern Lisp dialect aimed at practical scripting and tooling: tiny imperative core, familiar semantics, and a standard library you can learn in a day. It compiles to self-contained native binaries, embeds easily into C or even websites, and excels at text processing via PEG parsers and an expressive subprocess DSL. Powerful macros and compile-time execution blur build/runtime boundaries. HN readers like the design and art-tool examples, but note weak package/versioning, limited libraries, and tooling gaps.

### Comment pulse
- Janet is delightful but ecosystem small: weak package versioning, limited web/HTTP libraries; tools like joy and jeep help—counterpoint: gaps still deter production use.  
- Fennel’s Lua-based design yields speed and portability, but transpilation makes debugging fragile and unpleasant; Janet’s integrated VM, REPL, and debugger feel far more robust.  
- Some dislike Lisp-y square brackets and misstatements like equating SETQ with def; others defend bracket roles, note alternatives without them, and request stronger editor/LSP tooling.  

### LLM perspective
- View: Tiny core, metaprogramming, and easy distribution position Janet for bespoke tools, CLIs, DSL-heavy apps more than generic backend services.  
- Impact: If ecosystem matures, Janet could siphon scripting, game-embedding, and automation niches from Lua, Python, Bash where binary distribution matters.  
- Watch next: Better package management, HTTP/server frameworks, language server support, and more flagship apps would raise Janet adoption and seriousness.
