# Anthropic, please ship an official Claude Desktop for Linux

- Score: 436 | [HN](https://news.ycombinator.com/item?id=48434436) | Link: https://github.com/anthropics/claude-code/issues/65697

### TL;DR
The GitHub issue argues Anthropic should either ship an official Claude Desktop for Linux (Debian/Ubuntu LTS) or clearly state it’s not planned and why. Today, Linux users rely on unofficial repackages of the Windows Electron app to access key features like Cowork, desktop extensions, and secure credential handling, despite Anthropic already distributing signed Linux binaries for Claude Code and even running Cowork inside an Ubuntu VM. HN discussion centers on Linux fragmentation, support costs, real security risks, and the value gap vs the CLI.

---

### Comment pulse
- Linux desktop support is a compatibility minefield → distro/WM/driver fragmentation turns tiny user shares into disproportionate support load and angry, vocal users — counterpoint: Flatpak/AppImage help constrain the surface.  
- Many proprietary Linux GUIs are already Electron → users argue a large, AI-focused company should manage what one maintainer and other firms (Discord, Spotify, VS Code) already do.  
- Desktop app offers real advantages over CLI → Cowork, scheduled routines, cross-conversation search, rich UX, and workflow parity with non-Linux coworkers matter beyond coding-only terminals.
