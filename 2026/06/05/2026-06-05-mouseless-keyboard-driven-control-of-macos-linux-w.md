# Mouseless – keyboard-driven control of macOS/Linux/Windows

- Score: 430 | [HN](https://news.ycombinator.com/item?id=48383667) | Link: https://mouseless.click

### TL;DR
Mouseless is a cross-platform app that lets you control the mouse entirely from the keyboard, using typed “coordinates” and key-driven cursor actions (move, click, drag, scroll). It targets speed, ergonomics (reducing mouse strain/RSI), and accessibility for people who struggle with pointing devices. Hacker News discussion compares it with tools like Shortcat, Homerow, Keynav, Vimium-style extensions, and QMK mouse keys, and debates whether keyboard-based mouse emulation is a clever workaround or a symptom of poor native keyboard/accessibility design in modern GUIs.

### Comment pulse
- Coordinate-based keyboard control vs hint-based tools → Some find Mouseless faster and more predictable than Shortcat’s dynamic hints; others prefer “Vimium for OS” models like Shortcat/Homerow.

- Mouse emulation vs true accessibility → Critics say keyboard-driven cursors should be a last resort; command palettes, semantic navigation, and proper shortcuts reduce cognitive load and work across hidden UI—counterpoint: platforms still optimize for the majority mouse users.

- Keyboard maximalism vs mouse mastery → Some argue better mouse skills and acceleration (plus aim trainers) can rival productivity; others say improving keyboard workflows and app choice beats mouse training, especially for RSI.
