# πFS

- Score: 957 | [HN](https://news.ycombinator.com/item?id=48480978) | Link: https://github.com/philipl/pifs

- TL;DR  
    - πfs is a joke FUSE filesystem that “stores” files by treating every possible byte sequence as somewhere inside the digits of π, relying on the conjecture that π is normal. Instead of data, it saves metadata: the byte offsets and lengths of each chunk within π. The project parodies magical compression schemes: information theory says the address of arbitrary data needs as many bits as the data itself. HN discussion branches into entropy, one‑time pads, LLMs as lossy language compressors, and historical “infinite compression” claims.

- Comment pulse  
    - Library‑of‑Babel‑style schemes teach that, for arbitrary data, an address needs as many bits as the data—counterpoint: LLMs achieve useful, lossy language compression.  
    - Commenters note πfs can’t beat entropy: for long files, the π index usually exceeds file size; byte‑by‑byte lookup underscores the joke.  
    - Other “data‑free” ideas appear: NSAfs (government‑paid storage), impractical global one‑time‑pad streams, Sloot’s mythical ultra‑compression, and classic write‑only‑memory gags.
