# On The <dl> (2021)

- Score: 337 | [HN](https://news.ycombinator.com/item?id=48247325) | Link: https://benmyers.dev/blog/on-the-dl/

## TL;DR

The article explains HTML description lists: `<dl>` wraps a list of name–value groups, `<dt>` marks each “name,” and `<dd>` its “value(s)`. It shows how to use multiple `<dd>` per `<dt>`, optionally group each pair in a `<div>` for styling, and why this semantic pattern beats nested `<div>`s—especially for screen reader navigation and summarization. Examples range from product specs to Dungeons & Dragons stat blocks. HN discussion adds ARIA nuances, history, and pushback against “semantic purity.”

## Comment pulse

- `<dl aria-label="…">` is invalid without a compatible role → use `role="list"` (and `role="listitem"` on children), or drop `aria-label` — counterpoint: or avoid extra ARIA entirely.
- Semantic HTML feels too rigid → complex UIs need wrappers, icons, headings; devs fall back to `<div>` + ARIA; some screen reader users favor more pragmatic, JS-style APIs.
- DL/DT/DD predate the web via GML/SGML and appear on the first CERN site → long-standing pattern; richer semantics may also help code-understanding tools.
