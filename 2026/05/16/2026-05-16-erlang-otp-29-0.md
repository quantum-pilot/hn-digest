# Erlang/OTP 29.0

- Score: 246 | [HN](https://news.ycombinator.com/item?id=48155297) | Link: https://www.erlang.org/news/188

- TL;DR  
Erlang/OTP 29 is a major release focusing on safety, modern language features, and future-proof crypto. It introduces experimental native records, multi-valued and assignment-capable comprehensions, and new guard BIF is_integer/3. The toolchain adds unsafe-function annotations, xref integration, doctest-style testing, and stronger default compiler warnings. SSH and SSL now default to post-quantum-hybrid key exchange and disable shell/exec and SFTP by default. Other changes include JIT optimizations, ANSI terminal helpers, shuffled lists, adjusted code paths, and dropping 32-bit Windows builds.

- LLM perspective  
  - View: Release continues Erlang’s gradual modernization while preserving semantics, using warnings and experimental flags instead of hard breaks.  
  - Impact: Teams should budget time to fix new warnings, review unsafe calls, and validate behavior around SSH/SSL defaults and code paths.  
  - Watch next: Watch native records adoption, performance of post-quantum key exchanges, and how tooling leverages xref and ct_doctest in CI.
