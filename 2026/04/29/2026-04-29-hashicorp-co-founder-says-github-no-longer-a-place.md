# HashiCorp co-founder says GitHub 'no longer a place for serious work'

- Score: 391 | [HN](https://news.ycombinator.com/item?id=47946958) | Link: https://www.theregister.com/2026/04/29/mitchell_hashimoto_ghostty_quitting_github/

### TL;DR
Mitchell Hashimoto, HashiCorp co‑founder and early GitHub user (#1299), is moving his Ghostty terminal project off GitHub after weeks of near‑daily outages, especially in GitHub Actions, which he says now block “serious work.” He’ll leave a read‑only mirror but is actively migrating to another forge. Hacker News commenters report similar reliability pain, canceled GitHub migrations, and frustration with Microsoft’s AI‑heavy direction, yet also note that mainstream alternatives like GitLab have regressed, leaving teams feeling stuck.

---

### Comment pulse
- GitHub reliability is breaking workflows → teams report frequent Actions/PR outages, even canceling migrations from GitLab/CircleCI to GitHub.  
- Alternatives disappoint too → GitLab’s UX churn, bugs, and flaky self‑hosting erode trust; some prefer minimal tools like Azure Repos for being “too simple to break.”  
- Entrenchment is strong → most engineers barely care which forge they use, so mass departures seem unlikely—counterpoint: high‑profile exits may still push orgs to reassess.
