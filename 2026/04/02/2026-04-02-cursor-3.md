# Cursor 3

- Score: 258 | [HN](https://news.ycombinator.com/item?id=47618084) | Link: https://cursor.com/blog/cursor-3

### TL;DR
- Cursor 3 introduces a new agent‑centric interface on top of its VS Code–derived IDE: a unified “agents window” where multiple local and cloud agents work across repos in parallel. Sessions can be moved between desktop and cloud, agents generate demos/screenshots, and a new diff/PR view plus integrated browser and plugin marketplace aim to cover the whole path from code understanding to merged PR. HN discussion splits between excitement for richer agent workflows and concern about chat-first design, pricing, and real productivity gains.

### Comment pulse
- Many devs dislike "agent swarms" and chat-first UIs, preferring Cursor’s earlier code-first IDE model or plain VS Code + Claude Code.  
- Others welcome parallel agents to avoid idle time, or already run multi-agent setups via CLI and alternative tools like pi.dev.  
- Skepticism about Cursor’s business value and pricing: heavy users hit limits, expect first-party IDEs and open source to erode Cursor’s differentiation.

### LLM perspective
- View: Cursor 3 reframes coding as orchestrating many agents across repos, not editing files, pushing toward "vibe-first" development.  
- Impact: Teams with complex workflows may benefit most; solo devs content with one assistant might see cost and overhead increase.  
- Watch next: whether Cursor keeps robust code-first views, publishes solid productivity benchmarks, and clarifies pricing versus effectively reselling model tokens.
