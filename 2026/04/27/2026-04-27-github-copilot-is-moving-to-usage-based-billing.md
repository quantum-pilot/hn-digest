# GitHub Copilot is moving to usage-based billing

- Score: 515 | [HN](https://news.ycombinator.com/item?id=47923357) | Link: https://github.blog/news-insights/company-news/github-copilot-is-moving-to-usage-based-billing/

- TL;DR  
GitHub is replacing Copilot’s current “premium requests” with token-based GitHub AI Credits on June 1, 2026. Seat prices stay the same, but each plan now just includes equivalent-dollar credits, while chats/agents and code review burn tokens (and Actions minutes), with pooled budgets and previews for teams. Code completions remain free. HN commenters see this as the end of heavily subsidized inference: huge effective price hikes, especially for Anthropic models, and little reason for light users to stay over PAYG APIs.

- Comment pulse  
  - Subscription loses appeal → credits priced like APIs; unused monthly allotments waste money, so many plan to cancel and switch to PAYG providers.  
  - End of subsidized inference → heavy Copilot users burned hundreds of dollars of Opus/Sonnet tokens for $10; new per-token multipliers close that loophole.  
  - Enterprise-first strategy → many expect Copilot to focus on orgs needing integrated reviews, budgets, compliance—counterpoint: individuals can replicate features via IDE plugins and generic APIs.

- LLM perspective  
  - View: Usage-based billing makes sense economically but destroys flat-rate arbitrage; Copilot must now compete directly with raw API and local tools.  
  - Impact: Enterprises gain clearer budgeting and pooled credits; light or bursty individual users are incentivized to leave subscriptions for flexible providers.  
  - Watch next: Watch for Copilot adding unique, GitHub-native workflows or model deals; otherwise it risks becoming a middleman over commodity LLM APIs.
